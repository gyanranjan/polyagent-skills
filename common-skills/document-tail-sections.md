# Document Tail Sections

Standard sections to include at the end of generated documents for consistency and traceability.

## When to Apply

Apply these tail sections when a skill produces a formal document (requirement doc, implementation sketch, analysis report, etc.). Do NOT apply for informal outputs like email drafts or chat summaries.

## Sections

### Revision History

Include a revision history table at the end of every formal document:

```
| Version | Date       | Author | Changes          |
|---------|------------|--------|------------------|
| 1.0     | YYYY-MM-DD | [name] | Initial version  |
```

### Assumptions & Constraints

List any assumptions made during document creation and known constraints that bound the content.

### Open Questions

List unresolved questions that need stakeholder input, with suggested owners where possible.

### References

Link to source materials, related documents, and dependencies.

### Glossary (Optional)

Include if the document uses domain-specific terms that may not be universally understood.
