# Spec: Adapter Contract

**Author:** Gyan Ranjan
**Status:** Approved
**Created:** 2025-02-27
**Updated:** 2025-02-27

---

## Summary

Defines the contract that all agent adapters must follow. Adapters are thin translation layers between the portable skill library and agent-specific instruction formats.

## Motivation

Each agent discovers and reads instructions differently. Adapters bridge this gap. To keep the system maintainable, adapters must follow strict rules about what they can and cannot contain.

## Detailed Design

### The Golden Rule

> **Adapters contain pointers, never knowledge.**

An adapter tells the agent WHERE to find skills and HOW to read them. It never contains the skill logic itself.

### Adapter Directory Structure

```
adapters/<agent-name>/
├── <instruction-file>     # Agent's native instruction file
├── install.sh             # Copies adapter + skills into a target project
└── README.md              # Agent-specific notes
```

### Required Content in Every Adapter

1. **Skill discovery instructions** — Tell the agent that skills exist in `skills/`
2. **Skill consumption pattern** — Tell the agent to read `SKILL.md` in matching skill directories
3. **Common-skills pointer** — Tell the agent about `common-skills/`
4. **Available skill list** — List all skill names (auto-generated by `sync-adapters.sh`)

### Adapter Instruction File by Agent

| Agent | File | Format |
|-------|------|--------|
| Claude Code | `CLAUDE.md` | Markdown |
| OpenAI Codex | `AGENTS.md` | Markdown |
| AWS Kiro | `.kiro/specs/polyagent-skills.md` | Markdown |
| Google Gemini | `.gemini/instructions.md` | Markdown |
| Cursor | `.cursor/rules.md` | Markdown |
| Windsurf | `.windsurfrules` | Markdown |

### What Adapters MAY Contain

- Pointers to `skills/` and `common-skills/`
- Agent-specific quirks or workarounds (documented in Known Issues)
- Agent-specific configuration syntax required by the agent
- A list of available skill names and one-line descriptions

### What Adapters MUST NOT Contain

- Skill instructions or process steps
- Business logic or decision trees
- Duplicated content from `SKILL.md` files
- Hardcoded file paths (use relative paths)
- References to other agents

### Maximum Adapter Size

Adapters should be under 100 lines. If an adapter exceeds this, skill logic is leaking into it.

### install.sh Contract

Every adapter includes an `install.sh` that:

1. Accepts a target project directory as argument
2. Copies the adapter's instruction file to the correct location
3. Copies `skills/` and `common-skills/` to the target
4. Checks for existing instruction files and backs them up
5. Is idempotent — safe to run multiple times

Note: this contract is for **project-local adapter installs**. Global machine-level setup scripts (for example, `scripts/install-global-all.sh`) are complementary and do not replace adapter-level project install behavior.

### Auto-Generation

`scripts/sync-adapters.sh` regenerates the skill list section of each adapter by scanning `skills/*/SKILL.md` frontmatter. Manual edits to the skill list section will be overwritten.

## Acceptance Criteria

- [ ] Each adapter is under 100 lines
- [ ] No skill logic present in any adapter
- [ ] `install.sh` backs up existing files before overwriting
- [ ] `sync-adapters.sh` correctly updates all adapters
- [ ] Each adapter has been tested with its target agent

## Out of Scope

- How individual agents parse their instruction files (agent vendor's responsibility)
- Skill format (see skill-format-spec.md)
